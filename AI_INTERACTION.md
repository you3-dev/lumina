# 依頼：スマホブラウザ用「ウォーターソートパズル（Water Sort Puzzle）」の作成

## 1. ゲーム概要
* 複数のチューブ（試験管）に入った液体を移し替え、すべてのチューブを「1色（1模様）のみ」で満たす、あるいは「空」にするパズルゲームです。
* 制限時間はなし。じっくり最短手数を考える「思考型」の設計にします。

## 2. コアルール（オーソドックス）
* **移動条件:** 1. チューブの最上層にある液体のみを移動できる。
  2. 移動先のチューブに空きスペースがある。
  3. 移動先の最上層が「同じ種類（色・模様）」である、または移動先が「空」である。
* **移動量:** 同じ種類が連続している層は、可能な限り一度にまとめて移動させる。
* **勝利条件:** すべての液体が整理され、各チューブが「同じ種類4ユニット」または「空」になること。

## 3. アクセシビリティとデザイン（色覚特性への配慮）
* **模様（パターン）の付与:** 各色に必ず固有の視覚的パターン（例：ドット、ストライプ、チェック、波線、十字など）をCSSやSVGで重ねてください。「色」がわからなくても「模様」で100%判別できるようにします。
* **視認性:** チューブの境界線と、液体の層の境界線をはっきりと描画してください。

## 4. UI/UX（モバイル最適化）
* **操作:** チューブを1回タップして選択（浮かび上がる演出）、別のチューブをタップして注ぐ。
* **アニメーション:** CSS Transition等を使用して、液体が移動する様子をスムーズに表現。
* **機能:** - 「1手戻る（Undo）」ボタン（必須）。
  - 「最初からやり直す（Reset）」ボタン。
  - 手数カウンター。
* **レスポンシブ:** スマホの縦画面で、チューブが2段（または1段）で綺麗に収まるレイアウト。

## 5. 技術仕様
* HTML/CSS/JavaScriptの1ファイル完結（外部ライブラリ不使用）。
* **ステージ生成:** - あらかじめ「完成された状態」を定義し、それをゲームのルールに従って「逆順にランダム移動」させることで、**必ず解ける問題**を自動生成してください。
  - 初期設定はチューブ数7本（中身あり5本、空2本）程度。
